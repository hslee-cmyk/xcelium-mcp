## Executive Summary

| Perspective | Content |
|-------------|---------|
| **Problem** | 현재 구조가 ncsim legacy에 최적화되어 UVM, Makefile, xrun direct 등 다른 환경에서 동작 불가. run script 형식, bridge 주입 방법, test name 전달, compile/sim 분리가 환경마다 다름 |

## 1. 배경: 환경별 차이

### 1.1 현재 지원 (ncsim legacy)

```
run_sim -test VENEZIA_TOP015 -gate post --
  └─ compile (xmvlog/xmelab) + simulate (xmsim) 일괄
  └─ MCP_INPUT_TCL env var로 bridge 주입
  └─ setup_rtl.tcl / setup_gate.tcl 로 probe 설정
```

### 1.2 UVM 환경

```
# compile (1회)
xrun -f filelist.f -elaborate -uvm -access +rwc

# simulate (매 test)
xmsim -input bridge.tcl top +UVM_TESTNAME=my_test +seed=random
```

- compile/simulate **분리**
- test name = **UVM class name** (`+UVM_TESTNAME`)
- seed, coverage 옵션
- setup TCL 대신 UVM sequence가 시뮬레이션 제어
- bridge 주입: `xmsim -input` 직접 또는 `xrun -input @xmsim`

### 1.3 Makefile 기반

```
make compile MODE=rtl
make sim TEST=my_test SEED=1234 COV=1
```

- Makefile target으로 추상화
- 내부적으로 xrun/xmsim 호출
- bridge 주입: `SIM_ARGS += -input $(BRIDGE_TCL)`

### 1.4 xrun direct

```
xrun -f sim.f -top top +define+TEST=my_test -input bridge.tcl -access +rwc
```

- 단일 명령으로 compile + simulate
- bridge 주입: `-input` 직접

### 1.5 환경별 차이 요약

| 항목 | ncsim legacy | UVM | Makefile | xrun direct |
|------|:----------:|:---:|:--------:|:-----------:|
| compile/sim 분리 | 일괄 | **분리** | **분리** | 일괄 |
| test 전달 | `-test {name}` | `+UVM_TESTNAME={name}` | `TEST={name}` | `+define+TEST={name}` |
| bridge 주입 | `MCP_INPUT_TCL` env | `xmsim -input` | `SIM_ARGS` | `-input` 직접 |
| seed | 없음 | `+seed={seed}` | `SEED={seed}` | `+seed={seed}` |
| coverage | 없음 | `-coverage all` | `COV=1` | `-coverage all` |
| setup TCL | probe 설정 | 없음/선택 | 없음/선택 | 없음/선택 |
| mode 선택 | `-gate/-ams` flag | filelist 분리 | `MODE=gate` | `-f gate.f` |

## 2. 목표

1. **Runner 추상화**: 모든 환경을 통일된 인터페이스로 실행
2. **Wrapper script 자동 생성**: 기존 환경의 run script/Makefile에 wrapper를 씌워서 MCP 연동
3. **Compile/Simulate 분리 지원**: compile 1회 → simulate N회 패턴
4. **Bridge 주입 방법 추상화**: 환경별 bridge 주입 메커니즘을 wrapper가 처리
5. **Test parameter 추상화**: test_name, seed, coverage, mode 등을 통일된 파라미터로

---

## 3. 설계

### 3.1 Runner 추상화 레이어

**통일된 인터페이스:**

```python
class RunnerInterface:
    """모든 환경이 구현해야 하는 추상 인터페이스."""

    async def compile(self, sim_mode: str = "rtl") -> str:
        """Compile/elaborate. 이미 컴파일되었으면 skip."""

    async def simulate(
        self,
        test_name: str,
        sim_mode: str = "rtl",
        bridge_tcl: str = "",
        setup_tcl: str = "",
        seed: str = "",
        coverage: bool = False,
        timeout: int = 600,
    ) -> str:
        """Run simulation. bridge_tcl이 있으면 bridge mode."""

    async def list_tests(self, pattern: str = "") -> list[str]:
        """사용 가능한 test 목록."""

    def get_shm_path(self, test_name: str = "") -> str:
        """SHM dump 경로."""
```

**환경별 구현:**

```python
class NcsimLegacyRunner(RunnerInterface):
    """기존 run_sim script 기반."""
    # compile + simulate 일괄, MCP_INPUT_TCL env

class UvmRunner(RunnerInterface):
    """UVM xrun/xmsim 환경."""
    # compile/simulate 분리, +UVM_TESTNAME, +seed

class MakefileRunner(RunnerInterface):
    """Makefile target 기반."""
    # make compile, make sim TEST=...

class XrunDirectRunner(RunnerInterface):
    """xrun 단일 명령."""
    # xrun -f sim.f -input bridge.tcl
```

### 3.2 Wrapper Script 자동 생성

**핵심 아이디어**: 기존 프로젝트의 run script/Makefile을 **수정하지 않고**, MCP wrapper가 감싸서 통일된 인터페이스 제공.

**Wrapper 구조:**

```
{sim_dir}/.mcp/
├── mcp_wrapper.sh          # 통일된 entry point
├── mcp_compile.sh          # compile wrapper (환경별 생성)
└── mcp_simulate.sh         # simulate wrapper (환경별 생성)
```

**`mcp_wrapper.sh` — 통일된 인터페이스:**

```bash
#!/bin/bash
# Auto-generated by sim_discover. Do not edit.
# Usage: mcp_wrapper.sh <action> [options]
#   actions: compile, simulate, list_tests
#
# simulate options:
#   --test <name>       Test name
#   --mode <rtl|gate|ams_rtl|ams_gate>
#   --bridge-tcl <path> MCP bridge TCL (bridge mode)
#   --setup-tcl <path>  Probe setup TCL
#   --seed <value>      Random seed
#   --coverage          Enable coverage
#   --timeout <sec>     Timeout

ACTION=$1; shift

case $ACTION in
  compile)
    source .mcp/mcp_compile.sh "$@"
    ;;
  simulate)
    source .mcp/mcp_simulate.sh "$@"
    ;;
  list_tests)
    source .mcp/mcp_list_tests.sh "$@"
    ;;
  *)
    echo "Unknown action: $ACTION"
    exit 1
    ;;
esac
```

**ncsim legacy용 `mcp_simulate.sh`:**

```bash
#!/bin/bash
# Wrapper for ncsim legacy run_sim
# Injects MCP_INPUT_TCL and translates unified args to run_sim format

BRIDGE_TCL="" SETUP_TCL="" TEST="" MODE="rtl" SEED="" COV=""
while [[ $# -gt 0 ]]; do
  case $1 in
    --test)       TEST="$2"; shift 2;;
    --mode)       MODE="$2"; shift 2;;
    --bridge-tcl) BRIDGE_TCL="$2"; shift 2;;
    --setup-tcl)  SETUP_TCL="$2"; shift 2;;
    --seed)       SEED="$2"; shift 2;;
    --coverage)   COV="1"; shift;;
    --timeout)    shift 2;;  # handled by caller
    *)            shift;;
  esac
done

# Bridge mode: inject MCP_INPUT_TCL
if [ -n "$BRIDGE_TCL" ]; then
  export MCP_INPUT_TCL="$BRIDGE_TCL"
  [ -n "$SETUP_TCL" ] && export MCP_SETUP_TCL="$SETUP_TCL"
fi

# Translate mode to run_sim flags
MODE_FLAGS=""
case $MODE in
  gate)     MODE_FLAGS="-gate post";;
  ams_rtl)  MODE_FLAGS="-ams";;
  ams_gate) MODE_FLAGS="-amsf -gate post";;
esac

# Execute original run_sim
exec ./run_sim -test "$TEST" $MODE_FLAGS --
```

**UVM용 `mcp_simulate.sh`:**

```bash
#!/bin/bash
# Wrapper for UVM xmsim
# Translates unified args to xmsim +UVM_TESTNAME format

BRIDGE_TCL="" TEST="" SEED="random" COV="" RUN_DIR="run"
while [[ $# -gt 0 ]]; do
  case $1 in
    --test)       TEST="$2"; shift 2;;
    --bridge-tcl) BRIDGE_TCL="$2"; shift 2;;
    --seed)       SEED="$2"; shift 2;;
    --coverage)   COV="-coverage all -covoverwrite"; shift;;
    *)            shift;;
  esac
done

cd "$RUN_DIR"

# Build xmsim args
XMSIM_ARGS="-64bit -messages"
[ -n "$BRIDGE_TCL" ] && XMSIM_ARGS="$XMSIM_ARGS -input $BRIDGE_TCL"
XMSIM_ARGS="$XMSIM_ARGS +UVM_TESTNAME=$TEST +seed=$SEED $COV"

exec xmsim $XMSIM_ARGS top
```

**Makefile용 `mcp_simulate.sh`:**

```bash
#!/bin/bash
# Wrapper for Makefile-based flow

TEST="" MODE="rtl" BRIDGE_TCL="" SEED="" COV=""
# ... parse args ...

MAKE_ARGS="TEST=$TEST MODE=$MODE"
[ -n "$BRIDGE_TCL" ] && MAKE_ARGS="$MAKE_ARGS BRIDGE_TCL=$BRIDGE_TCL"
[ -n "$SEED" ] && MAKE_ARGS="$MAKE_ARGS SEED=$SEED"
[ -n "$COV" ] && MAKE_ARGS="$MAKE_ARGS COV=1"

exec make sim $MAKE_ARGS
```

**`sim_start` v5 — wrapper 경유:**

```python
# 모든 환경에서 동일한 호출:
async def _start_bridge(sim_dir, config, test_name, setup_tcl, sim_mode, timeout):
    wrapper = f"{sim_dir}/{config['runner']['wrapper_dir']}/mcp_wrapper.sh"

    cmd = (
        f"cd {cwd} && "
        f"(nohup bash {wrapper} simulate "
        f"--test {test_name} "
        f"--mode {sim_mode} "
        f"--bridge-tcl {bridge_tcl} "
        f"--setup-tcl {setup_tcl} "
        f"{_build_redirect(log_file)} < /dev/null &)"
    )
```

**장점:**
- `sim_start` 코드가 **환경 무관** — wrapper가 환경별 차이를 흡수
- 기존 run script/Makefile **무수정** — wrapper가 감쌈
- 새 환경 추가 = wrapper template 추가만

## 5. 전체 워크플로우 (v5)

### 5.1 ncsim legacy (기존과 동일 UX)

```
sim_discover(sim_dir="/users/.../ncsim")
  → type: ncsim_legacy
  → wrapper 생성: .mcp/mcp_wrapper.sh
  → "Wrapper created. Ready."

sim_start(test_name="VENEZIA_TOP015_i2c_8bit_offset_test", sim_mode="rtl")
  → mcp_wrapper.sh simulate --test ... --mode rtl --bridge-tcl ...
  → xmsim 시작 + auto-connect
```

### 5.2 UVM (마이그레이션 후)

```
sim_discover(sim_dir="/users/.../uvm")
  → type: uvm
  → wrapper 생성: .mcp/mcp_wrapper.sh
  → "Wrapper created. Ready."

sim_start(test_name="i2c_reg_test", sim_mode="rtl")
  → mcp_wrapper.sh compile --mode rtl  (필요 시)
  → mcp_wrapper.sh simulate --test i2c_reg_test --bridge-tcl ... --seed random
  → xmsim 시작 + auto-connect

list_tests(pattern="i2c")
  → mcp_wrapper.sh list_tests --pattern i2c
  → ["i2c_reg_test", "i2c_stress_test", "i2c_error_inject_test"]
```

### 5.3 Gate simulation

```
sim_start(test_name="VENEZIA_TOP015_i2c_8bit_offset_test", sim_mode="gate")
  → mcp_wrapper.sh compile --mode gate  (SDF elaboration)
  → mcp_wrapper.sh simulate --test ... --mode gate --bridge-tcl ...
  → timeout=1800, probe_strategy=selective 자동 적용
```

## 6. v4.1과의 관계

| 항목 | v4.1 | v5 |
|------|------|-----|
| Runner | `runner.type` + `args_format` + `script_has_cd` | RunnerInterface + Wrapper |

**마이그레이션 경로:**
- v4.1의 `sim_start` → v5에서 wrapper 경유로 변경
- v4.1의 `sim_discover` → v5에서 wrapper 생성 추가
- v4.1의 multi-bridge, auto port → **그대로 유지**
- v4.1의 SimVision tool → **그대로 유지**
